# 2026-05-27 实验日志

## 目标

P3 Step3: GoFa CRB15000 机械臂关节空间轨迹跟踪控制

## 尝试的方案

### 方案1: Python PD + 固定重力补偿 (motor actuator)
- 原理: `torque = Kp*(target - q) - Kd*qvel + grav_comp`，grav_comp 在零位预计算
- 结果: 预热（保持零位）成功，轨迹跟踪立即发散（795°+ 误差）
- 根因: 固定重力补偿只在零位局部有效，关节角度变化后补偿错误

### 方案2: Python PD + 动态 qfrc_bias 补偿 (motor actuator)
- 原理: torque = KP*e - KD*qvel + data.qfrc_bias（每步动态计算）
- 结果: 预热阶段就发散（qpos 偏到 200°+），qfrc_bias 在偏离位置产生错误正反馈

### 方案3: MuJoCo `<position>` actuator + gear
- 原理: 使用 MuJoCo 官方内置的 position servo，`gear^2 * kp * (target - q)`
- 结果: QACC instability at DOF 4 (joint_5)，gear^2 放大导致远端轻量关节刚度太高

### 方案4: MuJoCo `<position>` actuator 无 gear
- 原理: 直接 joint-space 位置控制，`ctrl = target_angle_rad`
- 结果: 预热阶段 arm 掉到地面以下（tool0_z = -0.086m），position actuator 施加的力矩与预期不符

### 方案5: 高 KP + 固定重力补偿 (KP=3000, motor)
- 原理: 提高 PD 增益使 KP*e 压倒重力补偿误差
- 结果: 仍立即发散，高 KP 导致大幅过冲 → 高速运动 → 科氏力爆炸（qfrc_bias 飙升到 40000+ Nm）

### 方案6: 动态重力补偿（每步根据 target 计算 grav_comp）+ motor
- 原理: 保存状态→设 target→mj_forward→取 qfrc_bias→恢复状态→PD+补偿
- 结果: 仍立即发散，重力补偿在 target 位置与零位差异不够大，KP*target 无法压倒 g(target)

### 方案7: 计算力矩控制 (Computed Torque)
- 原理: `torque = M @ (KP*e - KD*qvel) + qfrc_bias`
- 结果: 仍然发散，M 矩阵从 qM packed format 提取不正确（mj_fullM 使用有误）

## 根本原因分析

1. **零位是不稳定平衡点**：机械臂垂直零位，重力使任何微小偏离被放大
2. **重力随关节角度剧变**：joint_2 从 0°→15°，重力从 -22.5 Nm 变为 -54.2 Nm；从 0°→-15°，重力翻转成 +10.8 Nm
3. **惯性耦合**：对 joint_1 施加力矩，通过惯性矩阵的 off-diagonal 在 joint_4 产生更大的加速度——分散 PD 控制根本上不适合多体系统
4. **MuJoCo API 理解不足**：qM packed 格式、mj_fullM 正确用法未掌握

## 关键教训

1. **先验证单关节再扩展多关节**，不要一步到位做 6-DOF 控制
2. **更换控制策略前先分析当前策略的根因**，不要反复调参（KP 从 300 到 3000 来回调是浪费时间）
3. **MuJoCo 的高级 API（qM, qfrc_bias）使用前需先验证理解正确**
4. **记录"官方优先"原则到 CLAUDE.md**（已完成）

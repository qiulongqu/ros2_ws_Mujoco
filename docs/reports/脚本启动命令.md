## 0. 通用前置

### Conda 环境

本项目使用 **两个独立环境**, 不可混用:

| 环境 | 用途 | 激活 |
|------|------|------|
| `mujoco` | 所有纯 MuJoCo 仿真 (P1~P4 tutorials) | `conda activate mujoco` |
| `/opt/ros/humble` (系统) | ROS2 + MoveIt2 + ros2_control | `source /opt/ros/humble/setup.bash` |

**混用后果**: ROS2 的 `PYTHONPATH` 与 MuJoCo 的 Python 3.13 环境冲突, 报 `ModuleNotFoundError` 或 DLL 加载失败.

## 5. ROS2 + MoveIt2 (项目顶层)

**目录**: `src/gofa_mujoco_bringup/` + `src/gofa_moveit_config/`

### 5.1 构建 + 加载环境

```bash
cd /home/yunhao2204/ros2_ws_abb/mujocoONLY

# 首次: 构建 + source
colcon build --symlink-install
source install/setup.bash
```

### 5.2 启动 MuJoCo + ros2_control (基础)

```bash
source /opt/ros/humble/setup.bash
source ~/ros2_ws_abb/mujocoONLY/install/setup.bash

# 有 GUI (打开 MuJoCo Simulate 窗口)
ros2 launch gofa_mujoco_bringup gofa_mujoco.launch.py

# 无 GUI (headless, 服务器/容器用)
ros2 launch gofa_mujoco_bringup gofa_mujoco.launch.py headless:=true

# 发送关节位置命令 (rad)
ros2 topic pub /position_controller/commands std_msgs/msg/Float64MultiArray \
  "{data: [0.5, -0.3, 0.2, 0, 0, 0]}"

# 监控关节状态
ros2 topic echo /joint_states
```

### 5.3 启动完整 MoveIt2 + MuJoCo + RViz (demo)

**P5 baseline 闭环 (2026-06-09)**: JTC + move_group 配置 + 启动验证全过, 详见 `p5_moveit2_learning/docs/troubleshooting/0001_JTC_blocking_point.md`.

```bash
source /opt/ros/humble/setup.bash
source ~/ros2_ws_abb/mujocoONLY/install/setup.bash

# 默认: MuJoCo + RViz 都开
ros2 launch gofa_moveit_config demo.launch.py

# 只开 RViz (关掉 MuJoCo 窗口, 跑 headless)
ros2 launch gofa_moveit_config demo.launch.py mujoco_gui:=false

# 只开 MuJoCo (关掉 RViz)
ros2 launch gofa_moveit_config demo.launch.py rviz_gui:=false

# 全 headless (服务器/容器用)
ros2 launch gofa_moveit_config demo.launch.py mujoco_gui:=false rviz_gui:=false
```

**控制器分流原则 (踩坑 2026-06-09, CLAUDE.md 5-06)**:
- `demo.launch.py` (P5) — 只 spawn `abb_controller` (JTC, 接 FollowJointTrajectory action)
- `gofa_mujoco_bringup/gofa_mujoco.launch.py` (P4) — 只 spawn `position_controller` (ForwardCommandController, topic pub)
- 两个 controller 都 claim `joint_X/position`, 同一进程不能并存, 必须用不同 launch 文件分流

**验证清单 (5 项必过)**:
```bash
# 1. controllers 状态
ros2 control list_controllers
# 期望: abb_controller active + joint_state_broadcaster active

# 2. /joint_states 50Hz 发布
ros2 topic hz /joint_states
# 期望: average rate: 50.x Hz

# 3. action server 在线
ros2 action list | grep trajectory
# 期望: /abb_controller/follow_joint_trajectory

# 4. controller 类型已注册
ros2 control list_controller_types | grep Trajectory
# 期望: joint_trajectory_controller/JointTrajectoryController

# 5. 启动 move_group (规划节点)
# 注意: move_group 假设 controllers 已 active, 不要在 controllers 未就绪时启动
ros2 launch gofa_moveit_config move_group.launch.py &
ros2 action list | grep -i move
# 期望: /move_action
```

## 6. 常用诊断命令

```bash
# 查看 ROS2 节点
ros2 node list
ros2 node info /ros2_control_node

# 查看所有 topic
ros2 topic list
ros2 topic info /joint_states

# 查看控制器
ros2 control list_controllers

# 加载 / 切换 / 卸载控制器
ros2 control load_controller joint_state_broadcaster
ros2 control switch --controllers joint_state_broadcaster position_controller
ros2 control unload_controller position_controller

# TF 树
ros2 run tf2_tools view_frames
```

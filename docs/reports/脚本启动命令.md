**P5 baseline 闭环 (2026-06-09)**: JTC + move_group 配置 + 启动验证全过, 详见 `p5_moveit2_learning/docs/troubleshooting/0001_JTC_blocking_point.md`.


**控制器分流原则 (踩坑 2026-06-09, CLAUDE.md 5-06)**:
- `demo.launch.py` (P5) — 只 spawn `abb_controller` (JTC, 接 FollowJointTrajectory action)
- `gofa_mujoco_bringup/gofa_mujoco.launch.py` (P4) — 只 spawn `position_controller` (ForwardCommandController, topic pub)
- 两个 controller 都 claim `joint_X/position`, 同一进程不能并存, 必须用不同 launch 文件分流

#### 5.4.6 故障排查 (端到端验证失败时)

| 症状 | 根因 | 修复 |
|------|------|------|
| `ModuleNotFoundError: No module named 'rclpy._rclpy_pybind11'` | 用了 conda python (3.13), rclpy 是 3.10 编译的 | 改用 `/usr/bin/python3` |
| `action /abb_controller/follow_joint_trajectory not available` | demo.launch.py 没起来, 或 JTC 没 spawn | 跑 §5.4.3 验证 1, 确认 `abb_controller active` |
| `JTC goal 被拒绝` | 关节名拼错, 或 joint_states 与 SRDF 不一致 | 检查 `gofa.srdf` group 定义 + 脚本里 `JOINT_NAMES` |
| `status: 4` 但 `终点误差 > 5°` | kp 太小, 或目标位姿在奇异点附近 | 改目标角度 (避开 ±90° 边界), 或调 kp (CLAUDE.md 3-01) |
| `timeout: result not received` | move_group.launch.py 没起, `/execute_trajectory` 没人接 | 跑 §5.4.3 验证 3, 确认 `/execute_trajectory` 在 action list 里 |
| `Could not identify parent group for end-effector 'gofa_gripper'` | SRDF end_effector 找不到 parent group (WARN, 不影响) | 可忽略, 是 MoveIt 内部警告, 不影响规划 |
| `MoveGroup context initialization complete` 后没看到 "You can start planning now!" | OMPL 加载失败, 或 SRDF 有错 | 删 build 重新 `colcon build --packages-select gofa_moveit_config` |

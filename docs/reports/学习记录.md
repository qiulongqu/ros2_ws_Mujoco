## Mujoco仿真的层级

### 类比: 你在学开车

想象你在开一辆自动挡的车:

- 导航 (L1 规划):     "去浦东机场"      → 给你路线
- 方向盘轨迹 (L2):    "前 500m 直行, 再右转"  → 路径曲线
- 自动驾驶 (L3 伺服):  "油门踩 30%, 方向盘转 15°" → 怎么开车
- 物理世界 (L4):      "路有坡, 风有阻力, 轮胎打滑" → 实际车速/位置

机械臂完全一样:

- IK (L1):     "末端要到 (0.5, 0.15, 0.5)"  → 算出 6 个关节角
- S-curve (L2): "从当前角到目标角, 用平滑曲线" → 一段时间内的目标序列
- PD 伺服 (L3):  "关节差 2° 就用力, 速度快就反向阻尼" → 输出力矩
- 物理引擎 (L4): "重力 + 关节连杆 + 碰撞下, 算下一步位置" → 实际 qpos

### L1 路径规划

**用一句话总结**

┌─────────┬──────────────────────────────────────────────────┬─────────────────────────────────────┐                          
│   层    │                      一句话                       │               谁来做                │ 
├─────────┼──────────────────────────────────────────────────┼─────────────────────────────────────┤
│ L1 规划 │ "末端到 XYZ"反解成"6 个关节角", 还要找无碰撞路径     │ Python 算 (本项目) / MoveIt2 (工业)  │
└─────────┴──────────────────────────────────────────────────┴─────────────────────────────────────┘

L1 输出: 一串离散路径点 (关节角序列) + 路径点之间的连接方式. 交给 L2 做时间参数化.

---

具体来讲，**L1 的核心 = 3 件事**:
- IK (Inverse Kinematics) — "末端到 XYZ, 关节该怎么转" (离散)
- 路径规划 (RRT*, PRM) — "找一条不撞桌子的路" (关节空间搜索)
- 轨迹优化 (抛物线/时间最优) — "路径上每点跑多快" (这其实跟 L2 联动)

再把3大子任务拆开讲：

**1.子任务: IK (逆运动学)**

问题: 已知末端 XYZ, 求 6 个关节角.

- 正向 (FK): 6 关节角 → 末端 XYZ         (唯一解, 闭式)
- 逆向 (IK): 末端 XYZ → 6 关节角          (可能有 0/1/多解)

其真正的输出就是六个关节的关节角，比如 q=[-16.5, -17, 73.3, 1.6, 10.1, 0]° (这里写角度值，但是实际工程中都用弧度制。)

**2.子任务: 路径规划 (RRT*)**

问题: 已知起点 q_start, 终点 q_goal (IK 算的), 中间有桌子, 找一条不撞桌子的关节空间路径.

实际应用场景里有不同的算法，这里列举几条常用的，这些算法的集合就叫**OMPL**（Open Motion Planning Library，MoveIt2 默认的规划后端）

1. RRT* 算法（OMPL默认）: 在 6 维空间撒点 + 生长树 + 渐进优化, 找到绕过桌子的路径（RRT的改版）
2. RRT-Connect 算法：在RRT*上拓展，使用了双向树拓展方法，从起点和终点交替生成树，加快连接的速度和准确性。
3. Ruckig 算法（MoveIt2 默认）：Ruckig就像一位"顶级赛车手"，能在机器人的物理极限（速度、加速度、急动度）约束下，算出让机械臂用最短时间完成动作的"最优驾驶方案"。

**注**：Ruckig 算法和上面两个不是同一级的，实际工作中由OMPL执行RRT/RRT-Connect，"找一条无碰撞的路"，然后Ruckig 处理 速度/加速度/急动度约束。

**3.子任务: 轨迹优化 (平滑性 + 限速)**

问题: 路径规划给出离散点 [q1, q2, q3, ...], 怎么让机械臂平稳通过?

做法:
- 路径点 → B 样条 / 三次样条插值
- 加速度/jerk 限制
- 这就是上一节讲的 L2 轨迹, 但 L1 也会做 (粗规划), L2 做精细规划

这里的**粗规划又涉及到 6 个关键参数 (工业经验值)**：

  ┌─────────────────┬─────────────┬──────────────┬────────────────────────────────────────────────┐
  │      参数       │     值      │     含义      │                    工业调参                    │
  ├─────────────────┼─────────────┼──────────────┼────────────────────────────────────────────────┤
  │ eps             │ 0.0005 rad  │ 有限差分步长  │ 太小 (1e-6) → 数值噪声; 太大 (1e-2) → 偏导不准   │
  ├─────────────────┼─────────────┼──────────────┼────────────────────────────────────────────────┤

### L2 轨迹（参数时间化）

**用一句话总结**
      
┌─────────┬──────────────────────────────────┬───────────────────────────────────┐                         
│   层    │              一句话               │              谁来做               │
├─────────┼──────────────────────────────────┼───────────────────────────────────┤
│ L2 轨迹 │ 把"从 A 到 B"变成"每秒在 C 位置"   │ Python 算 (你在 step2 / step3 写)  │
└─────────┴──────────────────────────────────┴───────────────────────────────────┘

L2 输出: 6 个关节的目标角度 数组, 500Hz 一次, 发给 L3 (PD 伺服). L3 不关心"路径", 只关心"我现在要到的位置".

**类比: 方向盘曲线和机械臂**

│  阶段   │              你 (司机)               │                          机械臂                         │
│ L2 轨迹 │ "前 5m 方向盘转 5°, 5-10m 转 10°..." │ "0.1s 关节到 2°, 0.2s 到 7.5°, 0.3s 到 15°..."           │

**M3的解释**：L2 的核心 = 时间参数化: 给"起点 + 终点 + 时长", 输出"每时刻在哪". 这就是 ptp_target(start, goal, t, duration) 这 1 个函数干的事.

**我的解释**：当你在L1层级给出了运动的目标点之后L2负责的就是每次（一般是500HZ，0.002秒）都计算下一步应该怎么走，把连续的移动过程拆分长多个不同的小的时间步，最后输出结果到L3层级。这种连续到离散的过程又可以叫做插值。

---

**4 种插值方式对比 (从粗暴到精细)**

问题来源：想象从 0° 到 60° 转 1.5 秒, 你能怎么走呢？

┌─────────────────────────┬─────────────────────────────┬────────────────────────┬──────────────┐
│          方式           │            公式             │      速度/加速度         │     适合     │
├─────────────────────────┼─────────────────────────────┼────────────────────────┼──────────────┤
│ ① 线性                  │ s = t/T                     │ 头尾速度突变, 有冲击     │ 别用         │
├─────────────────────────┼─────────────────────────────┼────────────────────────┼──────────────┤
│ ② 三次多项式             │ 3f²-2f³                     │ 头尾速度=0, 加速度突变   │ 入门         │
├─────────────────────────┼─────────────────────────────┼────────────────────────┼──────────────┤
│ ③ 五次多项式 (quintic)   │ 6f⁵-15f⁴+10f³               │ 头尾速度/加速度都=0     │ 本项目用的    │
├─────────────────────────┼─────────────────────────────┼────────────────────────┼──────────────┤
│ ④ 梯形/S-curve (工业级)  │ 多段拼接, 设最大速度/加速度   │ 受限                    │ MoveIt2 默认 │
└─────────────────────────┴─────────────────────────────┴────────────────────────┴──────────────┘

**注**：为什么 quintic 不够, 工业还要梯形/S-curve?

**答**：quintic *不限制最大速度/加速度*. 如果 goal 离 start 太远 (e.g. 180°), 中间速度会飙到天上, 关节电机跟不上就丢步.

### L3 伺服 (自动驾驶)

  ┌────────────────────────────────────┐
  │  PD 控制器 (MuJoCo 内置)            │
  │                                    │
  │  error = 目标(30°) - 实际(0°) = 30° │
  │  torque = kp × error - kv × speed  │
  │         = 4000 × 30° - 400 × 0     │
  │         = 120000 N·m (单位转换后)   │
  │                                    │
  │  → 输出 120 N·m 力矩给关节电机      │
  └────────────────────────────────────┘

其中的3 个参数用开车类比

  ┌────────────┬──────────┬────────────────────────────────────┐
  │    参数    │   含义   │              开车类比             │
  ├────────────┼──────────┼────────────────────────────────────┤
  │ kp=4000    │ 比例增益 │ 你踩油门的力度: 差越远, 油门越深    │
  ├────────────┼──────────┼────────────────────────────────────┤
  │ kv=400     │ 阻尼     │ 你松油门的时机: 速度快了, 反向轻踩  │

  └────────────┴──────────┴────────────────────────────────────┘

### L4 物理 (物理世界)

L3 输出的力矩, 进到 L4 物理引擎, 算下一步状态:

  ┌────────────────────────────────────────────┐
  │  物理引擎 (MuJoCo C 写的)                   │
  │                                            │
  │  已知:                                       │
  │    - 当前 6 个关节位置 qpos[6]               │
  │    - 当前 6 个关节速度 qvel[6]               │
  │    - 6 个质量块 (6 个连杆) 质量 + 惯量       │
  │    - 重力 g = 9.81 m/s²                      │
  │    - L3 给的 6 个力矩 ctrl[6]                │
  │                                            │
  │  算:                                        │
  │    - 重力对每个关节的力矩 (Σ 静力学)          │
  │    - 离心 + Coriolis 力 (Σ 运动学)           │
  │    - 6×6 质量矩阵 M(q)                       │
  │    - 解 M·q'' = τ_ctrl + τ_gravity + τ_coriolis  │
  │                                            │
  │  输出:                                       │
  │    - 下一步 qpos[6], qvel[6]                 │
  │    - 6 个连杆的 xpos[6], 末端 xpos           │
  └────────────────────────────────────────────┘

为什么需要 L4?

1. 没有 L4: 你说"关节 1 转到 30°", 它瞬间就 30°. 但实际上, 6 关节连杆有惯性, 还有重力往下拉, 真实位置和目标永远有差.
2. L3+L4 联动: L3 持续"加油", L4 持续"算实际位置", 两者每 0.002 秒 (500Hz) 算一次. 跟得上的话差几°, 跟不上差几度.

### 4个层级的串联循环

L1 给出目标的机械臂位姿

IK 算出目标关节角 (e.g. [30, -15, 0, 45, -30, 60]°)
                  ↓
L2 轨迹 (本项目)
    ptp_target(start, goal, t, duration)  ← 1 个函数
    ↳ 算出"在 t 时刻, 目标关节角是多少"
                  ↓
L3 伺服 (MuJoCo 内置 PD)
    torque = kp*(target - qpos) - kv*qvel
                  ↓
L4 物理 (MuJoCo 引擎)
    mj_step() → qpos 更新
                  ↓
(回到 L2 顶部, 下个 0.002s 循环)

## 调参实战 (工业上改什么)

**调参的本质**：

调参 = 在参数不准的情况下, 找到一组"皮实"的 kp/kv 让控制稳定.

***

为什么要调参：

  ┌──────────────────┬───────────┬───────────────────────────────┐
  │   想改善的现象   │  改哪里   │            怎么改             │
  ├──────────────────┼───────────┼───────────────────────────────┤
  │ 移动太慢         │ L3 kp     │ 调大 (4000 → 5000)            │
  ├──────────────────┼───────────┼───────────────────────────────┤
  │ 到目标后震荡     │ L3 kv     │ 调大 (kv/kp 保持 1/10)        │
  ├──────────────────┼───────────┼───────────────────────────────┤
  │ 高速段超过目标   │ L2 插值   │ 用 S-curve 替代梯形           │
  ├──────────────────┼───────────┼───────────────────────────────┤
  │ 跟踪误差大 (>2°) │ L4 积分器 │ integrator="implicit"         │
  ├──────────────────┼───────────┼───────────────────────────────┤
  │ 物理不稳定爆炸   │ L4 时间步 │ timestep 缩小 (0.002 → 0.001) │
  └──────────────────┴───────────┴───────────────────────────────┘

其中有一些本项目调参的经验总结：

- kv = kp / 10 不要破
- 高 kp + 显式 Euler 必震荡 → 换 integrator="implicit"
- kp 从近端 (大关节) 到远端 (小关节) 递减: 4000 → 4000 → 3000 → 2500 → 2000 → 1500
- 远端 (link_5/6) 质量小 (0.63kg / 0.14kg), kp 太大会震荡

## sim2real的真实环境最常见的问题

1: 标定误差

问题: 厂方 CAD 给的 mass=13.5kg, 实际机器称重可能是 14.2kg (含螺栓/润滑油/线缆).

量级:
- 工业机器人质量公差: ±5-10%
- 重心位置公差: ±1-3cm
- 惯量张量公差: ±10-30% (因为形状假设 ≠ 实际)

后果: 仿真稳态误差 0.4°, 真机可能 1-2°.

工业做法 (system identification):
1. 让机器人做特定轨迹 (e.g. 慢速圆弧)
2. 记录力矩 + 位置数据
3. 拟合质量/重心/惯量
4. 工具: sympybotics, robex, 或自写最小二乘

---

2: 温度漂移 (电机发热)

问题: 电机跑 1 小时后温度升 30°C, 摩擦系数变化 20-40%, 阻尼也变.

实测数据 (我从 GoFa 数据手册的逻辑推):
- 冷态 kp 稳态 0.5°
- 热态 (运行 1h) kp 稳态 1.2°  ← 摩擦增加
- 解决方案: 工业机器人内置温度传感器, 调参表里查表补偿

真实机器人: ABB OmniCore 控制器里有 thermal model, 自动补偿.

3: 减速器背隙 (backlash)

问题: 谐波减速器/ RV 减速器有微小间隙 (典型 0.1-0.5°), 换向时误差跳变.

仿真:
<joint damping="0.1" armature="0.01"/>
- damping: 阻尼, 仿真摩擦
- armature: 等效惯量 (电机侧 + 减速比² 反映到连杆), 仿真减速器影响

但没仿真 backlash 间隙! 真机换向时, 电机转 0.3° 连杆完全没动, 然后突然跟上.

后果: 高精度轨迹 (e.g. 装配) 误差 0.3-1°. 真机没办法纯靠 PD 解决, 需要加前馈 + 间隙补偿.

4: 未建模动力学 (缆线柔性 + 关节间隙)

问题: 真实机械臂的:
- 关节 O 型圈/轴承有粘滑摩擦 (stiction), 低速时表现诡异
- 缆线 (走线) 走 6 个关节累计长度 2-3m, 高速时缆线弹动会引入振动
- 减速器本身有扭转刚度, 大惯量负载下会"扭"

仿真:
- 缆线: 完全没模型
- 粘滑摩擦: Coulomb friction 也只能近似
- 减速器刚度: 默认刚性 (不仿真扭转)

真实工程: 这些效应在高速高加速 (e.g. 激光切割) 时明显, 低速不影响.

5: 负载变化 (抓不同物体)

问题: 仿真里方块 0.1kg, 真实可能抓 0.5kg / 2kg / 5kg / 10kg 物体, 动态完全不同.

工业做法 (CLAUDE.md P3 提到的):

┌─────────────────────┬────────────────────────────────────────────────┐
│      负载范围       │                    调参策略                    │

│ 空载 (无负载)       │ kp/kv 用空载值                                 │

├─────────────────────┼────────────────────────────────────────────────┤
│ 未知负载 (随机)     │ 加自适应控制 (自适应 PD) 或观测器 (扩展卡尔曼)    │
└─────────────────────┴────────────────────────────────────────────────┘
